# セクションマーク & 活用機能 設計

## コンセプト

zen-scrapを「rawデータの蓄積 + 外に繋げるハブ」にする。セクション単位でマークして取り出せる仕組みを加え、学びの活用を加速させる。

zen-scrapの責務はメモの蓄積とデータを外に渡しやすくすること。AI問題生成・採点・間隔反復などは外部ツールに委ねる。

## 機能一覧

### 機能1: セクションマーク

- 各セクションにブックマークトグルボタンを追加
- ON/OFFの1種類
- マーク状態はセクション見出しに `[marked]` として保存

### 機能2: セクション単位コピー

- 各セクションにコピーボタンを追加
- Markdownテキストをそのままクリップボードにコピー
- 既存のスクラップ全体JSONコピーとは別の機能

### 機能3: スクラップ内マーク絞り込み

- スクラップ詳細画面のヘッダーに「マーク済みのみ表示」トグルを追加
- ONにするとタイムラインがマーク済みセクションだけに絞り込まれる
- アウトラインドロップダウンも連動して絞り込まれる
- OFFに戻すと全セクション表示に復帰

### 機能4: マーク一覧ページ

- スクラップ一覧のツールバーに「マーク一覧」ボタンを追加
- 押すとマーク一覧ページに遷移（スクラップ一覧とは別ページ）
- 全スクラップ横断でマーク済みセクションをフラット表示
- 各セクションの表示内容: 冒頭プレビュー + 元スクラップ名 + 日時
- セクションごとにコピーボタン
- クリックで元スクラップの該当セクションに遷移
- 複数セクションを選択してまとめてコピー可能
- 「戻る」でスクラップ一覧に戻る

## データ構造

### ScrapEntryの変更

```typescript
interface ScrapEntry {
  timestamp: string;
  body: string;
  marked: boolean;  // 追加
}
```

### Markdownでの保存形式

```markdown
---
title: インプットログ
status: open
tags: [react, frontend]
created: 2026-03-10T14:00:00.000Z
updated: 2026-03-10T14:30:00.000Z
archived: false
pinned: false
---

### 2026-03-10 14:00 [marked]

ReactのuseMemoは参照等価性を保つための...

---

### 2026-03-10 14:30

通常のエントリ（マークなし）

---
```

セクション見出しの末尾に `[marked]` を付与する。パーサーがこれを読み書きする。

## 画面構成

```
スクラップ一覧
  ├── スクラップ詳細 → マーク / コピー / 絞り込み
  └── マーク一覧 → プレビュー / コピー → スクラップ詳細へ遷移
```

## ユースケース

1. 「今日の学び」をまとめる - マーク一覧から今日マークしたものを集めてコピー、AIに整理を頼む
2. テーマ別に深掘りする - 複数スクラップからマークしたセクションをまとめてコピー、AIに分析を頼む
3. 復習したいものを取り出す - マーク一覧からコピーしてAnkiやAIで問題を作る
4. スクラップ内の要点確認 - マーク絞り込みで要点だけざっと見返す
5. 調査の結論を拾う - マーク絞り込みで結論セクションだけ素早くたどる

## スコープ外

- AI問題生成・採点（外部ツールの役割）
- 間隔反復・進捗管理（外部ツールの役割）
- テンプレート付きコピー（まずはプレーンコピーで始める）
- セクション単位のタグ付け（将来の拡張候補）
