# ミニマップ & URL自動埋め込み 設計ドキュメント

## 背景

zen-scrapを月次スクラップとして運用する場合、エントリ数が増えてスクラップが長くなる。
長いスクラップ内の情報に素早くアクセスする手段と、URLを貼るだけで埋め込みになる仕組みがあると、収集・閲覧の体験が大幅に改善される。

## 機能A: ミニマップ

### 概要

詳細ビューの右側にCanvas描画のミニマップを表示する。VSCodeのミニマップと同様に、タイムライン全体を縮小して描画し、現在のビューポート位置をハイライトする。クリック・ドラッグでスクロールジャンプできる。

### レイアウト

```
┌─────────────────────────────────┬────┐
│                                 │ ▓▓ │ ← ビューポートハイライト
│  タイムライン（既存）             │ ░░ │ ← ミニマップ（Canvas）
│                                 │ ░░ │
│                                 │ ░░ │
│                                 │    │
├─────────────────────────────────┴────┤
│  入力エリア（既存）                    │
└─────────────────────────────────────┘
```

- ミニマップ幅: 80〜100px程度
- タイムラインコンテナの右端にposition: absoluteで配置
- タイムラインの横幅はミニマップ分だけ縮む

### Canvas描画

タイムラインのDOMを走査し、テキストノードの位置・サイズをCanvasにマッピングする。

- テキスト行 → グレーの細い矩形（文字は描画しない。色の濃淡でテキスト密度を表現）
- 埋め込みブロック → 種類ごとに色分けした矩形（YouTube=赤、カード=青、GitHub=緑、Tweet=水色）
- セパレータ → 薄い水平線
- タイムスタンプ見出し → やや太い矩形

### スクロール連動

- メインスクロール → ミニマップ上のビューポート矩形が追従
- ミニマップクリック → 対応位置にメインがスクロール
- ビューポート矩形ドラッグ → メインがリアルタイムにスクロール

### 描画タイミング

- 初回: 詳細ビュー表示時
- 再描画: エントリ追加・編集・削除時、ウィンドウリサイズ時
- debounce付きで不要な再描画を抑制

### トグル

- ヘッダー部にON/OFFボタン配置
- ショートカット: Cmd/Ctrl + M
- 表示状態はセッション内で保持（デフォルト非表示）

### エッジケース

- エントリ0件: 空のCanvas表示
- エントリが非常に多い: 最小行高さ1px、溢れた分はミニマップ自体もスクロール可能
- 全画面表示モード: 同じ位置に表示（特別な処理不要）

### 影響範囲

- 新規: src/ui/minimap-renderer.ts
- 変更: scrap-detail-view.ts（ミニマップ領域配置、トグルボタン）
- 変更: header-renderer.ts（トグルボタンUI）
- 変更: timeline-renderer.ts（再描画トリガー発火）

## 機能B: URL自動埋め込み

### 概要

textareaにURLをペーストすると、自動で埋め込み記法に変換する。設定でON/OFFを切り替え可能（Inkdrop方式）。

### データフロー

```
textarea paste event
  → clipboardData.getData("text/plain")
  → テキスト全体が単一URLか判定
  → YES → URLパターン判定（YouTube / X / GitHub / その他）
        → 設定ON → 埋め込み記法に変換して挿入
        → 設定OFF → URLそのまま挿入
  → NO → テキストそのまま挿入（変換しない）
```

### URLパターン判定

既存のembed-modal.tsにある判定ロジックを共用関数として切り出す。

- YouTube: youtu.be/*, youtube.com/watch?v=*, youtube.com/embed/*
- X/Twitter: twitter.com/*/status/*, x.com/*/status/*
- GitHub: github.com/*/*/blob/*（コード行リンク）
- その他のURL: https://で始まる → @[card](URL)

### 変換ルール

- テキスト全体が単一URL → 埋め込み記法に変換
- テキストの中にURLが混在 → 変換しない
- 既に埋め込み記法（@[で始まる）→ 二重変換しない

### 設定

settings.tsに autoEmbed: boolean（デフォルト: true）を追加。

### 影響範囲

- 変更: input-area-renderer.ts（pasteイベントリスナー追加）
- 変更: embed-modal.ts（URL判定ロジックを共用関数に切り出し）
- 変更: settings.ts（autoEmbed設定追加）

## テスト方針

手動テストを前提とする。

- URL自動埋め込み: YouTube 3パターン、X、GitHub、一般URLのペーストで正しい記法に変換されること
- 設定OFF時にURLがそのまま貼り付けられること
- ミニマップ: エントリ追加・削除で再描画されること
- ミニマップ: クリック・ドラッグでスクロールが連動すること

## アプローチ選定の経緯

### ミニマップ

Canvas描画型を採用。DOM縮小クローン型はDOMが倍になりパフォーマンス懸念があり、Obsidianプラグインの文脈では避けたい。月次スクラップが長くなる前提ではCanvas描画の軽量さが活きる。

### URL自動埋め込み

Inkdrop方式（設定ON/OFF）を採用。確認モーダル方式は毎回の操作が煩わしく、Slack方式（自動変換+Undo）は複数URL時のUI設計が複雑になる。設定1つで制御する割り切りが実装もUXもシンプル。
