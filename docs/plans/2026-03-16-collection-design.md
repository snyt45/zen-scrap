# コレクション機能 設計ドキュメント

## 背景

zen-scrapは学習の生ログを雑に蓄積するツールとして機能している。スクラップが増えると知識の断片が複数スクラップに散らばり、後から特定テーマでまとめて取り出すことが難しくなる。

ユーザーは自分でまとめ作業をしたくない。外部AI（Claude等）に材料を渡して整形してもらう運用を想定している。zen-scrapはデータのハブであり、AI機能は組み込まない。

## 目的

複数スクラップからエントリやスクラップを目的別に集めて、AIに渡せる形でコピーできる仕組みを提供する。

## データモデル

### Collection

```typescript
interface Collection {
  id: string;           // ユニークID（タイムスタンプベース）
  title: string;        // "TypeScript Tips" など
  items: CollectionItem[];
  created: string;      // ISO 8601
  updated: string;      // ISO 8601
}
```

### CollectionItem

```typescript
interface CollectionItem {
  type: "scrap" | "entry";
  scrapPath: string;        // スクラップのファイルパス
  entryTimestamp?: string;  // type="entry"のとき、エントリのタイムスタンプ
  order: number;            // 並び順
}
```

### 保存先

`{scrapsFolder}/collections.json` に全コレクションをまとめて保存する。スクラップのMarkdownファイルとは完全に分離。元データに一切影響しない。

## UI構成

### タブ追加

既存のタブナビゲーションに「コレクション」タブを追加する。

```
[一覧] [マーク] [コレクション] [+ 新規作成]
```

### コレクション一覧

- コレクションのリスト表示（タイトル、アイテム件数、作成日）
- 「+ 新しいコレクション」ボタン
- コレクションの削除

### コレクション詳細

- タイトル表示・編集
- アイテム一覧（スクラップ/エントリのプレビュー表示）
- アイテムのドラッグ並べ替え
- アイテムの削除（元データに影響しない）
- アイテムクリックで元のスクラップ/エントリにジャンプ
- 「+ 追加」ボタン → 検索モーダル
- 「まとめてコピー」ボタン

### 検索モーダル（+ 追加）

- スクラップ名での検索
- 追加モードの選択: スクラップ全体 / エントリを選んで追加
- スクラップ選択後、エントリモードならエントリ一覧を表示して選択

### 既存メニューへの追加導線

- スクラップ一覧のアイテムメニューに「コレクションに追加」
- エントリのメニューに「コレクションに追加」
- 押すとコレクション選択のドロップダウンを表示

## コピー形式

「まとめてコピー」で生成されるテキスト形式:

```markdown
# TypeScript Tips

## TS学習ログ（スクラップ全体）
description:
TypeScriptの型システムを理解する

### 2026-03-12 10:30
Partial<T>を使うとすべてのプロパティがオプショナルになる。

### 2026-03-12 11:00
Record<string, number>で動的キーの型が書ける

---

## Go並行処理メモ > 2026-03-14 15:00（エントリ単体）
goroutineのリーク防止パターン
ctx.Done()でキャンセルを受け取る

---
```

- コレクションタイトル → h1
- スクラップ全体 → スクラップタイトルがh2、descriptionとエントリが続く
- エントリ単体 → 「スクラップ名 > タイムスタンプ」がh2、本文が続く
- アイテム間は `---` で区切り

## データ整合性

- スクラップ/エントリが削除された場合: コレクション内の参照アイテムはグレーアウト表示。自動削除しない
- コレクションを削除した場合: 元のスクラップ/エントリには一切影響しない
- コレクションからアイテムを除外した場合: 参照が消えるだけ。元データはそのまま
- 整合性チェックのタイミング: コレクション詳細を開いたとき

## スコープ外

- AI組み込み（zen-scrapはハブ。AIは外部で利用）
- Canvas的な空間配置
- マーク機能の廃止（コレクションと共存）

## 既存機能との関係

- マーク機能: 共存。マークはスクラップ内でエントリに印をつける瞬間的な行為。コレクションは目的を持って横断的に集める行為。役割が異なる
- JSONコピー: 既存のまま。コレクションのコピーはAI向けに最適化された別形式
- タグ/検索: 既存のまま。コレクションの検索モーダルで活用
